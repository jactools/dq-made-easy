# Docker Image Landscape - Current State (2026-07-09)

## Executive Summary

The dq-made-easy system consists of **22+ actively maintained Docker images** organized around:
- **Core platform services** (API, Engine, Frontend, Database, Auth)
- **Infrastructure services** (Kong gateway, Kafka, Redis, Keycloak)
- **Metadata & discovery** (OpenMetadata stack)
- **Observability stack** (Prometheus, Grafana, Loki, Tempo, OTEL Collector)
- **Optional add-ons** (LLM, Trino, Airflow, Zammad support, Edge routing)

**Current Major Versions**: API/UI `0.11.5`, Core services `0.9-xxxxxxx` (git short SHA)

## Build & Publish Strategy

### Active Build Scripts

All services use individual build-and-push scripts in their respective directories:
- `dq-api/scripts/build_and_push.sh`
- `dq-base/scripts/build_and_push.sh`
- `dq-db/scripts/build_and_push.sh`
- `dq-engine/scripts/build_and_push.sh`
- `dq-ui/scripts/build_and_push.sh`
- `dq-keycloak/scripts/build_and_push.sh`
- `dq-kong/scripts/build_and_push.sh`
- `dq-profiling/scripts/build_and_push.sh`
- `dq-kafka/scripts/build_and_push.sh`
- (+ others for auxiliary services)

### Unified Build Orchestration

**`scripts/build_and_push_all.sh`** (v1.5) coordinates all builds:
- **Core scope** (default): 8 main services
- **Repo scope**: 22+ services including seed, metadata, observability, edge, airflow
- **Multi-arch support**: `linux/amd64,linux/arm64` by default
- **Version tagging**: Git short SHA or manual override
- **Platform**: BuildX for multi-platform publishing

### Docker Hub Publishing

- **Registry**: Docker Hub (configured via env variables)
- **Namespace**: Usually `jaccloud/` or registry-specific
- **Platforms**: Multi-arch manifests for core images

## TLS/SSL Strategy

All services use **mkcert** for self-signed certificates in development:
- Root CA: `tmp/certs/mkcert-rootCA.pem`
- Per-service certificates: `tmp/certs/services/<service>/`
- Trust bundle: `tmp/certs/trust/internal-ca-bundle.pem`
- Read-only mounts into containers

Production deployments use real certificates with infrastructure providers (K8s, ACA).
